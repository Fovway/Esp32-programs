# Прошивка №1: Часы для ESP32 + OLED 0.96" (SSD1306 128x64)

Эта прошивка показывает:
- текущее время (крупно, по центру),
- текущую дату (ниже времени),
- статус Wi‑Fi и синхронизации NTP в верхней строке,
- заглушки температуры и влажности (до подключения реального датчика).

При старте устройство:
1. подключается к Wi‑Fi,
2. применяет сохранённый часовой пояс,
3. синхронизирует время через NTP,
4. поднимает веб-страницу настроек часового пояса (с паролем).

## Подключение дисплея (I2C)

Для типового модуля SSD1306:
- `GND` -> `GND` ESP32
- `VCC` -> `3.3V` ESP32
- `SCL` -> `GPIO 4`
- `SDA` -> `GPIO 3`

Эти пины уже зафиксированы в прошивке:
`Wire.begin(3, 4);`

## Что нужно установить в Arduino IDE

1. Плату **ESP32 by Espressif Systems**.
2. Библиотеки:
   - `Adafruit GFX Library`
   - `Adafruit SSD1306`

(`WiFi.h`, `time.h`, `WebServer.h`, `Preferences.h` входят в пакет ESP32.)

## Настройка

Откройте файл `clock_oled.ino` и укажите Wi‑Fi:

```cpp
const char* WIFI_SSID = "YOUR_WIFI_SSID";
const char* WIFI_PASSWORD = "YOUR_WIFI_PASSWORD";
```

## Настройка часового пояса через ВЕБ

После подключения ESP32 к вашей сети откройте в браузере IP устройства:

- узнайте IP через Serial Monitor (можно добавить `Serial.println(WiFi.localIP());`),
- откройте `http://<IP_ESP32>/`.

Доступ к веб-странице защищён **паролем (Basic Auth)**:

```cpp
const char* WEB_AUTH_USER = "admin";
const char* WEB_AUTH_PASSWORD = "clock123";
```

На странице можно задать:
- `GMT offset` в секундах,
- `DST offset` в секундах.

Пример для Москвы:
- `GMT = 10800`
- `DST = 0`

Сохранённые значения записываются во flash (`Preferences`) и применяются после перезапуска.

## Температура и влажность

Сейчас в прошивке добавлены **заглушки** температуры/влажности:
- на экране выводятся меняющиеся тестовые значения `T` и `H`,
- это временно, пока физический датчик не подключён.

Когда появится датчик (например DHT11/DHT22/SHT3x), можно заменить функцию заглушки на чтение с реального сенсора без изменения остального UI.

## Идея по внешнему виду

Интерфейс сделан «чистым» и читаемым для 128x64:
- сверху компактная статусная строка,
- в центре крупные часы,
- внизу дата,
- мигающее двоеточие добавляет эффект «живых» часов.
